Частые вопросы

Всё, что нужно знать о демонстрационном экзамене группе 822

Подготовительный день

Вам дадут ~1 час за компьютером (по расписанию: 10:00–11:00 или 13:00–14:00 в зависимости от смены). В это время:

  • Расскажут, как и что будет проходить
  • Заведут за ваши рабочие места
  • Можно будет подключить сетевой диск
  • Можно пользоваться телефоном — это единственный день
  • Можно написать себе шаблоны и заготовки кода
Важно: Всё, что напишете в первый день, сохранится на компьютере и будет доступно во второй день. Компьютеры не чистят.

НЕТ

Задание дадут только во второй день (день экзамена).

В первый день вы готовите универсальные шаблоны: структура БД, авторизация, PDO-подключение, базовая вёрстка, слайдер, тест-кейсы.

Совет: Подготовьте файлы db.php, auth.php, шаблон подключения PDO, базовый HTML-каркас — это сэкономит много времени.

ДЕНЬ 1 ДА

В первый день телефон не запрещён. Можно:

  • Подсматривать синтаксис
  • Переписывать код с телефона на компьютер
  • Готовить шаблоны и заготовки

Интернета на компьютерах нет, поэтому ничего скачать не получится — только переписывать вручную.

БОНУСНЫЕ БАЛЛЫ

Да! Git даёт дополнительные баллы. Нужно сделать минимум 3 коммита.

Если планируете использовать Git — попросите установить его в первый день. Во второй день уже не получится.

Минимум коммитов: создание структуры → реализация БД → финальная версия. Это легко и даёт лишние баллы.

🎯 День экзамена

ВАЖНО

Регистрация начинается в 9:00. Явка строго без опозданий. Опоздавшие не допускаются к экзамену.

Рекомендуем приходить за 15-20 минут до начала.

ОБЯЗАТЕЛЬНО

Паспорт — без него не допустят к экзамену.

Больше ничего брать не нужно. Все материалы и оборудование предоставляются на месте.

Телефон брать нельзя! Даже в кармане. Забирают — даже если не пользуетесь.

ТРЕБОВАНИЕ

Деловой стиль: белый верх, чёрный низ.

  • Рубашка / блузка / водолазка (светлая)
  • Брюки / юбка (тёмные)
  • Закрытая обувь
Спортивные костюмы, шорты, яркие цвета — нельзя.

Общее время выполнения задания — 6 часов:

  • 9:15 - 11:15 — выполнение задания (2 часа)
  • 11:15 - 11:30 — технический перерыв (15 минут)
  • 11:30 - 13:30 — выполнение задания (2 часа)
  • 13:30 - 14:00 — обед (30 минут)
  • 14:00 - 15:00 — выполнение задания (1 час)
  • 15:00 - 17:00 — работа экспертов
Про перерывы: многие не следят за временем. Подходите к кабинету за 2 минуты до конца перерыва — вас пустят. Пару минут могут быть важны.

Обычно 3 человека, из которых ходят по аудитории только двое. Иногда они выходят из кабинета.

Они смотрят демонстрации экранов и проверяют, что у вас есть на компьютере.

Важно: не привлекайте к себе лишнее внимание, но и не нервничайте — эксперты настроены помочь, а не «завалить».

ТЕХНИЧЕСКОЕ

Все файлы проекта должны находиться в специальной папке на сетевом диске.

  • Сетевой диск подключаете в первый день и заново во второй
  • Вам покажут, какая это папка
  • На листочке будет ссылка на ваш сайт и ссылка на БД

Сам сайт вы не запускаете — вам дадут готовую ссылку для проверки.

Да, выходить можно сколько угодно в специально отведённое помещение (кабинет 313) для:

  • Приёма лекарств
  • Приёма пищи
  • Туалета

МАТЕРИАЛЫ

Все необходимые материалы уже находятся на компьютере:

  • Задание (PDF/текст)
  • Стили и шрифты (2 шрифта на выбор)
  • Тексты, фразы, адреса для сайта
  • Картинки и медиа
  • Справочник по HTML, CSS, Bootstrap
  • VS Code с базовыми функциями

Вам нужно только выбрать правильный шрифт из двух предложенных по заданию.

PDF может не открыться — переводите в Word заранее, если нужно посмотреть шрифты. Картинки PNG тоже могут открываться в браузере и блокироваться.

🛠 Техническое задание

ТЕХНИЧЕСКАЯ ЧАСТЬ

Любой билет = две страницы на PHP + MySQL:

  • Страница 1 — список/каталог/лента (с фильтром или поиском)
  • Страница 2 — детальная страница + форма действия (запись, заказ, комментарий и т.д.)

Пример: билет №4 — Каталог курсов с фильтром по направлению + Страница курса с кнопкой записи.

БАЗА ДАННЫХ

Во всех билетах 3 обязательные таблицы:

Таблица Назначение
users Регистрация и авторизация
products / articles / tours / courses / doctors... Тематическая (по заданию)
cart / comments / bookings / orders / requests... Таблица действий

Обязательно указать: поля таблиц, внешние ключи, связи (например, cart.user_id → users.id).

ER-диаграмма: можно сделать скриншотом прямо из phpMyAdmin — это засчитывается. Обязательно приложить к отчёту.

ОБЯЗАТЕЛЬНО

Да, во всех билетах. Требования:

  • Регистрация с password_hash()
  • Авторизация с password_verify() и session_start()
  • Проверка прав: неавторизованный пользователь не может выполнить действие
Важно: В разных билетах условия доступа разные. Например, в одних — неавторизованный видит описание, но не может слушать; в других — только авторизованный видит контакты.

КРИТИЧНО

Три обязательных пункта:

  • Все запросы к БД — через PDO::prepare() / execute()
  • Весь вывод в HTML — через htmlspecialchars()
  • Данные из $_POST никогда не выводятся напрямую
// Пример безопасного запроса
$stmt = $pdo->prepare("SELECT * FROM products WHERE category = ?");
$stmt->execute([$category]);

// Пример безопасного вывода
echo htmlspecialchars($product['title']);
Совет: Если забудете htmlspecialchars() — это минус баллы за безопасность.

SEO

Проверяют перед сдачей. Нужно сделать на каждой странице:

  • Уникальный <title>
  • Уникальный <meta name="description">
  • Только один <h1> на страницу
  • Семантические теги: <header>, <main>, <nav>, <footer>
  • Атрибут alt у всех <img>
Частая ошибка: забыть alt у картинок или поставить несколько <h1>.

ВЁРСТКА

Да, нужен адаптив. Обязательный media-query — на 390px.

@media (max-width: 390px) {
    /* ваши стили для мобильных */
}
Из опыта: Даже с недоделанным адаптивом можно получить 5. Главное — сделать хоть какие-то медиа-запросы.

ДА

В задании дадут два шрифта — нужно выбрать правильный по контексту и подключить.

Шрифты уже находятся на компьютере — скачивать ничего не нужно.

Если PDF со шрифтами не открывается — конвертируйте в Word, иначе не сможете посмотреть, какие шрифты использовать.

Зависит от конкретного билета. Возможные доп. задания:

  • Постраничная навигация (пагинация)
  • Карусель / слайдер (CSS Flexbox/Grid + JS)
  • Параллакс-эффект
  • Калькулятор с анимацией
  • Прогресс-бар
  • Аудиоплеер (только для авторизованных)
  • Валидация полей (например, только числа)
Про слайдер: Можно взять из Bootstrap (подключается по ссылке прямо в код). Даже если не знаете, как писать слайдер вручную — Bootstrap-карусель засчитывается. Один студент сделал только слайдер и таблицы — получил 5.

ОБЯЗАТЕЛЬНО

Да! Нужно сделать 3 тест-кейса:

  • Несколько позитивных (нормальное поведение)
  • Хотя бы 1 негативный (ошибочный сценарий)

Формат — такой же файл с таблицами, как на подготовке.

Желательно, но не обязательно для получения хорошей оценки.

  • Минимум: HTML + CSS + соблюдение всех требований задания
  • Оптимально: добавить авторизацию и регистрацию
  • Максимум: полностью рабочий сайт с бэкендом
Из опыта: Студенты, сделавшие только HTML+CSS с соблюдением требований, получали 5. Главное — соблюсти требования задания и сделать ER-диаграмму + тест-кейсы.

Правила поведения

ДЕНЬ 1 ДА — можно использовать для подготовки шаблонов.

ДЕНЬ 2 НЕТ — строго запрещено.

Во второй день: телефон нельзя брать вообще никак — даже в кармане. Забирают даже если не пользуетесь. После экзамена возвращают.
Совет: В первый день подготовьте все нужные шаблоны кода — во второй день телефон не понадобится.

НЕ РЕКОМЕНДУЕТСЯ

Теоретически можно попробовать пронести, но это риск.

Реальный случай: одного студента (Яну) поймали с бумажными шпаргалками — забрали. Не стоит рисковать.

Лучше запомнить основные синтаксические конструкции и подготовить шаблоны в первый день.

НЕТ

Нет интернета — скачивать ничего не получится. Плагины тоже нельзя.

Используйте только встроенные функции VS Code и справочник (HTML, CSS, Bootstrap уже на компьютере).

НЕТ

Взаимодействие с другими участниками, экспертами и посторонними лицами запрещено, если это не предусмотрено заданием.

Можно задавать вопросы только техническому эксперту по вопросам эксплуатации оборудования.

Оценивание и результаты

Работу проверяют 4 эксперта.

  • 5 (отлично): от 80 баллов
  • 4 (хорошо): от 60 баллов
  • 3 (удовлетворительно): от 40 баллов
Из опыта: «У меня CSS с недоделанным адаптивом — всё равно поставили 5 (84 балла). Главное — соблюсти требования задания.» Даже студент, сделавший только слайдер и таблицы, получил 5.

Оценки известны примерно за месяц до официальной даты. Если вы хорошо выполнили задание, оценку не занизят.

В группе, где у половины не работала БД или её вообще не было, не было ни одной двойки и тройки — всем натянули минимум 4.

Даже если сделали только HTML+CSS — это уже хорошо для пятёрки, если соблюдены все требования.

  • Минимум для 5: соблюдение требований + ER-диаграмма + тест-кейсы + вёрстка
  • PHP — желателен, но не критичен
  • Не пытайтесь сделать всё идеально — лучше сделать что-то простое, но полное
Важно: Не делайте только 1 файл. Нужно хотя бы 5 файлов для нормальной оценки.
